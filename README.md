## 简介

夜海的多用模组管理工具（NightNightOcean's Mods Manager）是一款 **多用 Mod 管理工具**。它采用 **插件驱动架构**，不绑定任何特定游戏，而是通过游戏插件来支持各种游戏。简单来说：

- 一个管理器就能管理多款游戏的 Mod
- 社区可以为任意游戏开发支持插件
- 功能可以靠插件不断扩展

### 能做什么？

本工具专注于 **Mod 的管理与组织**：

- **查看信息** - 方便地查看 Mod 名称、版本、作者、依赖关系等信息（需插件正确解析元数据）
- **问题检查** - 检测缺失依赖、加载顺序错误、重复 Mod 等问题
- **排序调整** - 拖拽或智能排序来调整 Mod 加载顺序
- **配置管理** - 保存多套 Mod 配置方案，随时切换

### 不能做什么？

作为多用管理器，它有一些局限：

- **不负责加载 Mod** - Mod 的实际加载由游戏完成，本工具只生成配置文件
- **不提供 Mod 下载** - 没有 Mod 下载功能，需要通过 Steam 创意工坊或其他途径获取 Mod
- **深度功能有限** - 对于某些游戏的深度定制功能（比如特有的 Mod 冲突解决机制），可能比不上专门为该游戏设计的管理器

### 适合什么样的 Mod？

本工具适合管理 **非侵入式 Mod**，即那些有独立目录、有明确标识的 Mod。具体来说：

- **有独立的文件夹** - 每个 Mod 都有自己的目录，不会直接覆盖游戏文件
- **有元数据文件** - Mod 自带描述文件，包含名称、版本、依赖等信息
- **有唯一标识** - 可以通过 ID 或文件夹名区分不同 Mod

典型的例子：

| 游戏             | Mod 元数据文件       |
| -------------- | --------------- |
| 环世界 (RimWorld) | `About.xml`     |
| 骑马与砍杀II：霸主     | `SubModule.xml` |
| 剑士 (Kenshi)    | `*.info` 文件     |

对于那种直接覆盖游戏文件的"侵入式 Mod"，本工具无法很好地管理。

> 💡 **小建议**：如果你只管理一款游戏的 Mod，而且这款游戏已经有专用管理器，那专用管理器可能功能更全面。但如果你同时玩多款 Mod 游戏，或者你喜欢的游戏还没有专用管理器，那本工具会是个不错的选择。*（如果有人开发插件）*

### 核心优势

本管理器最大的优势是 **不绑定任何游戏**。如果你的游戏没有好用的专用 Mod 管理器，可以通过开发一个简单的游戏插件快速实现基本的 Mod 管理功能：

- **基础插件只需 400-500 行 Python 代码** - 实现基本的 Mod 元数据解析和管理
- **一些进阶功能也不复杂** - 比如边缘世界插件包含存档读取功能，也只用了约 800 行代码

> 💡 **AI 友好开发**：本项目本身就是和 AI 合作开发的，所以插件开发也非常适合交给 AI 来完成。你只需要告诉 AI 你的游戏 Mod 结构，AI 就能帮你生成一个可用的游戏插件。

通过游戏插件，本工具可以支持多种游戏：

| 示例游戏          | Steam App ID | 备注          |

**想支持新游戏？** 写个游戏插件就行，不用改主程序。

- **双列表布局** - 左边未启用，右边已启用，一目了然
- **拖拽排序** - 拖一拖就能调整加载顺序
- **依赖关系可视化** - 用线条显示 Mod 之间的依赖关系
- **信息面板** - 查看 Mod 详细信息、标签、备注等等内容

- **智能排序** - 按 Mod 声明的依赖关系自动调整顺序（仅供参考，不保证完全正确）
- **问题检查** - 一键检测缺失依赖、顺序错误等问题

- **备注功能** - 给 Mod 加备注，记录心得

## 插件系统

插件系统是本工具的核心特色。

### 为什么要用插件架构？

传统的 Mod 管理器通常只支持一款游戏，功能也是固定的。而 NightOcean's Mods Manager 用插件驱动：

```
┌─────────────────────────────────────────────────────────┐
│                 NightOcean's Mods Manager               │
│                       (主程序核心)                       │
├─────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────┐  │
│  │ 游戏插件 A  │  │ 游戏插件 B  │  │   游戏插件 C    │  │
│  └─────────────┘  └─────────────┘  └─────────────────┘  │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────┐  │
│  │ 功能插件 X  │  │ 功能插件 Y  │  │   功能插件 Z    │  │
│  └─────────────┘  └─────────────┘  └─────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

### 双插件体系

| 插件类型     | 说明                  | 互斥性        |
| -------- | ------------------- | ---------- |
| **游戏插件** | 为特定游戏提供 Mod 解析和启动功能 | 同一时间只能启用一个 |
| **功能插件** | 扩展程序功能              | 可以同时启用多个   |

### 插件能做什么？

**游戏插件必须实现：**

- 解析游戏元数据和 Mod 元数据
- 提供游戏启动功能（本地启动、Steam 启动）
- 定义游戏特定的路径验证规则

**所有插件都可以：**

- **添加菜单项** - 在菜单栏加自己的操作
- **添加工具栏按钮** - 快速访问插件功能
- **添加自定义面板** - 展示插件特有的界面
- **订阅事件** - 响应 Mod 列表变化、游戏切换等事件
- **自定义高亮规则** - 按条件高亮显示 Mod
- **自定义过滤规则** - 实现复杂的筛选逻辑
- **独立配置存储** - 保存插件自己的设置
- **扩展错误检测** - 添加游戏特定的 Mod 问题检测
- **扩展右键菜单** - 在 Mod 列表右键菜单添加操作

**游戏插件还可以：**

- 解析存档文件（从存档导入 Mod 配置）
- 解析外部配置文件
- 自定义拓扑排序逻辑

### SDK 化开发

插件开发完全 SDK 化，开发者不需要拿到主程序源码。SDK 会随 Release 一并发布，下载后本地安装即可：

```bash
pip install yh_mods_manager_sdk-x.x.x-py3-none-any.whl
```

一个简单的功能插件例子：

```python
from yh_mods_manager_sdk import FeaturePlugin, PluginMenuItem

class MyPlugin(FeaturePlugin):
    PLUGIN_ID = "my_plugin"
    PLUGIN_NAME = "My Plugin"
    PLUGIN_VERSION = "1.0.0"
    
    def get_menu_items(self):
        return [
            PluginMenuItem(id="action", label="My Action", action_id="do_it")
        ]
    
    def on_menu_action(self, action_id, manager_collection):
        if action_id == "do_it":
            print("Action triggered!")
```

## 快速开始

程序已打包为一键运行的独立可执行文件，无需配置 Python 环境。

### 运行程序

1. 下载对应平台的可执行文件
2. 双击运行即可

### 首次使用

1. **安装游戏插件** - 程序启动后会提示安装游戏插件，选择你要管理的游戏对应的插件
2. **设置路径** - 指定游戏安装目录和 Mod 目录（程序会尝试自动检测）
3. **扫描 Mod** - 程序自动扫描可用 Mod
4. **开始管理** - 启用、排序、管理你的 Mod

## 开发者指南

### 项目结构

```
YHModsManager/
├── core/           # 核心业务逻辑
├── ui/             # 用户界面
├── plugin_system/  # 插件系统
├── utils/          # 工具函数
├── config/         # 配置文件
├── docs/           # 文档
└── yh_mods_manager_sdk/  # 插件 SDK
```

### 核心架构

项目采用分层架构：

- **UI 层** - 界面展示与用户交互
- **业务逻辑层** - Mod 操作、依赖解析
- **数据管理层** - 配置、元数据管理
- **插件系统层** - 插件加载与调度
- **基础设施层** - 日志、事件总线等
